Straffic final 중간발표 대본


안녕하십니까 KDT 선도기업과정 종로 1팀 최종 팀프로젝트 중간발표를 맡은 팀장 배영환입니다.


“발표는 팀 소개와 문제 정의를 시작으로,
프로젝트 개요, 시스템 아키텍처, 핵심 기능, DB 설계,
그리고 개발 과정에서의 이슈와 향후 계획 순으로 진행하겠습니다.”

“먼저 저희가 해결하고자 했던 문제를 말씀드리겠습니다.”

“현재 교통 시스템을 보면
CCTV 감시, 단속, 교통 흐름 관리, 설비 모니터링이
각각 따로 운영되고 있는 경우가 많습니다.”

“이로 인해
실시간 대응이 어렵고,
데이터가 서로 연결되지 않는 한계가 있습니다.”

“이 문제를 해결하기 위해 저희는 3인 팀으로 프로젝트를 진행했습니다.”

“저는 PM으로서 전체 설계와 대시보드, CCTV 단속,
그리고 AI 서버 및 OCR 기반 번호판 인식을 담당했습니다.”

“김소연 팀원은 UI/UX와 DB 설계 및 테스트를,
하재영 팀원은 실시간 데이터 수집과 인증 시스템, 예지보전 AI를 담당했습니다.”

“저희는 6개월 동안 학습한 웹 풀스택과 AI 기술을 바탕으로
이 문제를 하나의 방향으로 해결하고자 했습니다.”

“즉, 개별적으로 동작하던 시스템을
하나의 흐름으로 통합하는 것이 목표였습니다.”

“그 결과 구현한 시스템이 바로
AIpass, 통합 교통 관제 시스템입니다.”



“그러면 저희가 구현한 AIpass가 어떤 시스템인지 설명드리겠습니다.”

“AIpass는 교차로에 설치된 CCTV 영상을 AI가 실시간으로 분석하여,
과속 차량을 자동으로 감지하고, 번호판을 인식해
단속 정보를 시스템에 자동으로 저장하는
통합 교통 관제 시스템입니다.”

👉 (여기서 1초 멈춤 추천)


“저희는 단순한 데모가 아니라,
실제 환경을 기준으로 시스템을 검증하고자 했습니다.”

“그래서 타겟으로 선정한 지역이 강화도입니다.”

“강화도에는 ITS에서 제공하는 CCTV가 약 12개가
일렬 구조로 배치되어 있습니다.”

“이 구조는
여러 노선이 복잡하게 얽힌 도심과 달리,
흐름이 단순하기 때문에”

“AI 성능 측정과 시스템 검증을 위한
최소 기능 제품, 즉 MVP 환경으로 적합하다고 판단했습니다.”


“즉, 복잡도를 낮추고
핵심 기능에 집중하기 위한 전략적 선택이었습니다.”

“시스템 구조는 MSA 아키텍처를 기반으로 설계했습니다.”

“프론트엔드는 Vue 3,
백엔드는 Spring Boot,
AI 서버는 FastAPI로 구성하여
세 영역을 완전히 분리했습니다.”


“왜 MSA인가”는 짧고 강하게

“이렇게 분리한 이유는 명확합니다.”

“AI 처리, 특히 YOLO 객체 감지나 OCR은
연산 비용이 매우 크기 때문에”

“웹 서버와 분리하지 않으면
전체 서비스 성능에 영향을 주기 때문입니다.”


“AI 서버를 독립시킴으로써
각 서버가 자신의 역할에 집중할 수 있고,
일부 장애가 발생하더라도
전체 시스템은 안정적으로 유지되도록 설계했습니다.”

“저희가 정의한 MVP는
이 시스템이 end-to-end로 동작하는 것입니다.”

“전체 과정은 네 단계로 구성됩니다.”

“먼저, 강화도 CCTV 영상을 실시간으로 수신합니다.”

“다음으로, YOLO 모델이 차량을 감지하고
호모그래피 기반 알고리즘으로 속도를 계산합니다.”

“이후, 번호판 이미지를 OCR로 인식하고”

“마지막으로, 위반 정보가 백엔드를 통해
DB에 자동으로 저장됩니다.”

“다만 실제 CCTV 화질의 한계로 인해
현재는 일부 구간에서 샘플 이미지를 활용하고 있으며,
이 부분은 이후 이슈 파트에서 설명드리겠습니다.”

“이 모든 과정은 사람의 개입 없이
자동으로 처리됩니다.”

“현재 이 MVP 파이프라인은
실제로 동작 가능한 수준까지 구현된 상태입니다.”

“현재 전체 프로젝트 진행률은 약 80%이며,
남은 기간 동안 예지보전 AI 고도화와
통계 기능을 중심으로 개선할 예정입니다.”

“이제 실제로 시스템이 어떻게 구성되어 있는지
아키텍처를 통해 설명드리겠습니다.”





“이제 AIpass의 전체 시스템 구조를 설명드리겠습니다.”

“저희 시스템은 기능별 역할을 분리한
세 개의 독립적인 서버 구조로 설계되어 있습니다.”

“먼저, 프론트엔드는 사용자가 직접 보는 화면을 담당하며
Vue 3 기반으로 구현했습니다.”

“백엔드는 Spring Boot 기반으로
비즈니스 로직 처리와 DB 연동을 담당합니다.”

“마지막으로 AI 서버는 FastAPI 기반으로,
객체 감지와 속도 계산, 번호판 인식 등
모든 AI 처리를 전담합니다.”

“즉, 화면, 로직, AI 처리를 완전히 분리한 구조입니다.”

---
“이제 실제 데이터가 어떻게 흐르는지 말씀드리겠습니다.”

“사용자가 브라우저에서 시스템에 접속하면,
프론트엔드는 백엔드 서버에 API 요청을 보냅니다.”

“한편 AI 서버는
CCTV 영상을 실시간으로 받아 차량을 감지하고 속도를 계산합니다.”

“그리고 위반이 발생하면
해당 정보를 Webhook 방식으로
백엔드 서버에 자동으로 전달합니다.”

“백엔드는 이 데이터를 DB에 저장하고,
프론트엔드는 저장된 정보를 다시 조회하여
사용자 화면에 표시하게 됩니다.”

“이 과정을 통해
‘감지 → 분석 → 저장 → 시각화’가
하나의 흐름으로 연결됩니다.”

---
“이 구조에서 중요한 점은
각 서버가 서로 독립적으로 동작한다는 것입니다.”

“특히 AI 연산은 부하가 크기 때문에
별도의 서버로 분리함으로써
전체 시스템 성능과 안정성을 확보했습니다.”

“즉, 특정 기능에 문제가 발생하더라도
전체 서비스에는 영향을 최소화할 수 있도록 설계했습니다.”

---
“오늘 시연은 이 구조를 실제로 확인하실 수 있도록
두 대의 컴퓨터로 진행됩니다.”

“AI 서버는 제 환경에서,
프론트엔드와 백엔드는 시연 환경에서 각각 실행됩니다.”

“이는 물리적으로 분리된 환경에서도
시스템이 정상적으로 연동된다는 것을
직접 보여드리기 위함입니다.”

“그럼 지금부터
이 구조 위에서 동작하는 핵심 기능들을
순서대로 시연해 드리겠습니다.”






“핵심 기능 첫 번째, 인증 시스템입니다.”

“이 시스템은 일반 사용자가 아닌
관제 담당자를 위한 시스템이기 때문에
접근 제어가 매우 중요합니다.”

“따라서 로그인 기반으로
인증된 사용자만 시스템에 접근할 수 있도록 구성했습니다.”

“회원가입 시에는 이메일 인증을 통해
사용자 유효성을 검증하고,”

“로그인, 아이디 찾기, 비밀번호 재설정까지
기본적인 인증 흐름을 모두 구현했습니다.”

“인증은 HttpSession 기반으로 구현하였으며,
서버에서 세션을 관리하고”

“Interceptor를 통해 인증되지 않은 요청은
자동으로 로그인 페이지로 리디렉션됩니다.”

“이를 통해
인가되지 않은 접근을 차단하고,
시스템의 안정성을 확보했습니다.”







“다음은 대시보드 화면입니다.”

“대시보드는 관제 담당자가 가장 먼저 확인하는 화면으로,
현재 교차로 상황을 한눈에 파악할 수 있도록 설계했습니다.”

“관제 시스템에서 중요한 것은 데이터 양이 아니라
판단 속도라고 생각했습니다.”

“화면 상단에는 카카오맵 기반의 지도가 표시되며,
강화도 교차로 위치를 마커로 확인할 수 있습니다.”

“각 CCTV 마커를 클릭하면
해당 교차로의 실시간 영상을 모달로 바로 확인할 수 있습니다.”

“즉, 별도의 화면 이동 없이
대시보드 내에서 즉시 상황 확인이 가능합니다.”

“이를 통해 관제 과정에서의 불필요한 이동을 줄이고,
운영 효율성을 높이고자 했습니다.”

“지도 하단에는 상행, 하행, 전체 방향 필터를 제공하여
특정 구간의 혼잡도만 선택적으로 확인할 수 있습니다.”

“이를 통해 필요한 정보에 빠르게 집중할 수 있습니다.”

“지도 아래에는 주요 정보를 요약한 위젯들이 배치되어 있습니다.”

“예를 들어, 누적 교통량 위젯에서는
오늘 교통량과 전일 대비 변화, 최근 흐름을 함께 제공합니다.”

“또한 CCTV 상태, 오늘 단속 건수, 날씨 정보 등을
실시간으로 확인할 수 있습니다.”

“모든 위젯은 30초, 1분, 5분 단위로 자동 갱신되며,
갱신 주기는 관제자가 직접 설정할 수 있습니다.”

“결과적으로 대시보드는
여러 정보를 개별적으로 확인하는 것이 아니라,
한 화면에서 빠르게 상황을 판단할 수 있도록 설계했습니다.”



“다음은 CCTV 모니터링 화면입니다.”

“강화도 ITS에서 제공하는 CCTV를
그리드 형태로 구성하여,
최대 12개 채널을 동시에 확인할 수 있습니다.”

“여러 CCTV를 동시에 확인하면서도,
필요한 화면은 즉시 확대해 볼 수 있도록 구성했습니다.”

"각 카드를 클릭하면 모달로 확대하여 상세 확인이 가능합니다."

“여기서 한 가지 고려해야 할 문제가 있었습니다.”

“여러 CCTV 영상을 동시에 스트리밍할 경우
브라우저에 상당한 부하가 발생한다는 점입니다.”

“이를 해결하기 위해
Intersection Observer를 적용했습니다.”

“화면에 실제로 보이는 CCTV에 대해서만
스트림을 활성화하고,”

“화면 밖으로 벗어난 영상은
자동으로 스트림을 해제하는 방식입니다.”

“이를 통해 불필요한 리소스 사용을 줄이고,
여러 채널을 동시에 표시하더라도
안정적인 성능을 유지할 수 있었습니다.”

“또한 모든 CCTV를 단순히 모니터링하는 것에 그치지 않고,”

“관리자가 특정 CCTV를 선택하여
AI 분석 대상으로 지정할 수 있습니다.”

“선택된 CCTV 영상은 AI 서버로 전달되어
과속 감지와 번호판 인식이 실시간으로 수행됩니다.”

“즉, CCTV 모니터링과 AI 분석이
하나의 흐름으로 연결되도록 설계했습니다.”
