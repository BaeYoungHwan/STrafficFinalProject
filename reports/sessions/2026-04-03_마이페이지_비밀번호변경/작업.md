## 작업 개요

마이페이지(`/mypage`) 비밀번호 변경 기능의 UI 개선 및 보안 강화 작업을 완료하였다.
화면정의서 기준에 맞춰 모달 방식에서 카드 내 인라인 전환 방식으로 변경하고,
비밀번호 검증 로직 및 버그를 수정하였다.

---

## 구현 내용

### 1. 비밀번호 변경 UI 방식 변경
- **변경 전:** 모달 팝업(별도 오버레이) 방식
- **변경 후:** 카드 내 전환(인라인 폼 토글) 방식
- **근거:** 화면정의서 ver 1.0 반영 — iOS 느낌의 카드 전환 UX

### 2. exitPwMode() 비밀번호 데이터 즉시 초기화
- 비밀번호 변경 모드 종료 시 입력 필드 데이터를 즉시 클리어
- **목적:** 메모리에 비밀번호 문자열이 잔류하지 않도록 보안 강화

### 3. 비밀번호 규칙 5가지 제출 시 전체 검증
- 프론트엔드: 제출 시점에 5가지 규칙 모두 통과 여부 확인
- 백엔드: Spring Boot MemberController에서 동일 규칙 재검증
- **목적:** 프론트-백 검증 동기화로 우회 공격 방어

### 4. 현재 비밀번호 필드 제거
- 화면정의서 기준에 따라 현재 비밀번호 입력 필드를 UI에서 제거
- **QA/보안 검토 결과:** 세션 인증(HttpSession) 유지 중이므로 인증 상태는 확인됨
  - 단, 현재 비밀번호 재확인 없이 변경 가능한 점은 보안 위험으로 인지
  - 발표 일정 우선으로 팀 결정 하에 진행 (추후 개선 권고)

### 5. Map.of("data", null) NPE 버그 수정
- `Map.of()`는 null 값을 허용하지 않아 NullPointerException 발생
- HashMap으로 교체하여 null 허용 처리

### 6. 미사용 CSS `.btn-submit` 제거
- MypageView.vue에서 더 이상 참조되지 않는 CSS 클래스 제거
- 코드 청결도 향상
